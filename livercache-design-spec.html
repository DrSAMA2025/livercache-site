01 · Rationale

Why LiverCache Matters

Publication bias leaves a graveyard of valuable hepatology findings invisible to the field. LiverCache changes that.

📊

Fights Publication Bias

Null, negative, and inconclusive results rarely get published — yet they're vital for preventing duplicated effort and wasted grant funding.

♻️

Reduces Research Waste

By making shelved work discoverable, LiverCache prevents other researchers from re-running the same failed experiments unknowingly.

🔍

Encourages Transparency

Open science demands honesty about what didn't work. LiverCache normalizes sharing the full picture of a research journey.

🏅

Provides Academic Credit

Every cached entry receives a DOI — giving contributors a citable, permanent record of their shelved work that counts toward their academic profile.

02 · Page Specifications

Website Structure

Three core pages. Each spec includes purpose, UI components, and a wireframe preview. Click a tab to explore each page.

Homepage

The front door of LiverCache. It should immediately communicate the project's mission, invite exploration, and provide a clear path to contribute. Tone: confident, scientific, welcoming.

  • Navigation Bar Logo left; links to Home, Browse, Submit, About, Contact. "Submit" styled as a prominent CTA button. Sticky on scroll.
  • Hero Section Large headline + tagline. Primary CTA: "Explore Research" → Browse page. Subtle background texture or gradient.
  • About the Project Short paragraph explaining LiverCache's mission as an open-access, community-driven hepatology archive.
  • Recent Submissions 4 cards showing newest entries. Each card: title, contributor name, content-type tag. Cards link to full submission page.
  • How to Contribute Short paragraph + secondary CTA button: "Submit Your Work" → Submit page.
  • Footer Logo, copyright, social links, nav links. Reused on all pages.

Browse Page

The core discovery interface. Researchers come here to search and filter the LiverCache archive. Must be fast, scannable, and require minimal cognitive load.

  • Header / Nav Same sticky navigation as homepage.
  • Search Bar Full-width search; allows keyword, title, or author queries. Prominent placement at top of content area.
  • Filter: Content Type Checkboxes: Article, Dataset, Protocol, Poster. Allow multi-select.
  • Filter: Discipline Dropdown: NAFLD/NASH, Hepatitis, Cirrhosis, Cholestatic Disease, Other.
  • Filter: Date Dropdown: Last 30 Days, Last 6 Months, Last Year, All Time.
  • Results List Each result: title (links to full page), contributor name, submission date, content-type tag, discipline tag. Clean list layout — no images.
  • Footer Same as homepage.

Submit Page

The contribution gateway. Researchers submit their shelved work here. The form should feel frictionless and trustworthy. A clear confirmation message appears after submit.

  • Header / Nav Same sticky navigation as homepage.
  • Intro Text "Submit Your Research" heading + short paragraph explaining the submission process and its value.
  • Title Text field. Required.
  • Contributor Name(s) Text field. Supports multiple names. Required.
  • Content Type Dropdown: Article, Dataset, Protocol, Poster, Other.
  • Discipline Dropdown: NAFLD/NASH, Hepatitis, Cirrhosis, Cholestatic Disease, Other.
  • Abstract / Summary Large text area. Required.
  • Main Content File upload button (PDF accepted) OR external link field. At least one required.
  • Related Links Text field + "Add another link" button. Optional.
  • Contact Email Text field. Required. Private — not displayed publicly.
  • Submit Button Primary CTA. On click: validate fields → show inline confirmation message. Implement via Google Forms embed or custom form handler.
  • Footer Same as homepage.
03 · Technical Architecture

Proposed Stack

Built entirely on free, open-source tools. Zero infrastructure cost at launch. Designed to scale with the community.

🐙

GitHub Organization

Core repository for version control, issue tracking, and governance documents. All content is version-controlled and forkable by the community.

github.com/livercache CONTRIBUTING.md GOVERNANCE.md MIT License
🌐

GitHub Pages

Static site hosting directly from the repository. Free, fast, and reliable. Supports custom domains. Content updated via pull requests.

Static HTML/CSS/JS Custom domain Auto-deploy on push
📋

Google Forms

Submission intake via embedded or linked Google Form. Responses stored in Google Sheets for lightweight curation workflow at launch.

Submission intake Google Sheets backend No server needed
📦

Zenodo / OSF

File hosting and DOI issuance for contributed research assets. Each submission gets a permanent, citable DOI. Linked from the LiverCache entry.

DOI issuance Permanent archival CC BY 4.0
📂

Entry Structure

Each "cached" item is a Markdown file in the repository. Structured frontmatter enables automated listing and filtering on the Browse page.

Markdown + frontmatter title / authors / date reason-iced vocabulary sensitive-data flag
⚖️

Governance

Clear open-source governance from Day 1. Contributor license agreements (DCO/CLA) ensure content can later be transferred to AASLD or EASL stewardship.

CC BY 4.0 (content) MIT (code) DCO / CLA AASLD pathway

05 · Launch Roadmap

From Idea to Archive

A phased approach that gets to a working archive fast, then scales with community adoption.

Week 1 · Now

Foundation

Set up GitHub organization, primary repository, LICENSE files, CONTRIBUTING.md, CODE_OF_CONDUCT.md, and GOVERNANCE.md. Stand up basic GitHub Pages site with placeholder content.

GitHub Org Governance Docs Static Site
Month 1

Soft Launch

Full 3-page site live. Submission form goes live via Google Forms. Announce via hepatology listservs, Twitter/X, LinkedIn, and AASLD network. Accept first batch of submissions.

3-Page Site Google Forms Social Launch
Month 3–6

First Cohort & DOIs

Curate first batch of reviewed entries. Issue first DOIs via Zenodo. Recruit 2–3 volunteer curators from the hepatology community. Begin tracking submission and visitor metrics.

First DOIs Volunteer Curators Zenodo Integration
Year 1

Evaluate & Scale

Assess impact (submissions, citations, community growth). Apply for small grants or society partnerships (AASLD, EASL). Explore GitHub Sponsors for sustainability. Consider dedicated domain and enhanced search.

Impact Report Grant Applications Society Partnerships
